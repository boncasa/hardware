Lesson 10

Áp dụng vào DS-02 mà vẫn an toàn

Với thiết bị TanDev/DS-02, mục tiêu học là hiểu máy đang sống thế nào trước khi thay não nó. Vì vậy thứ tự đúng là observe, backup, identify, rồi mới flash lab nhỏ.

Bài học lý thuyết

Thiết bị mua sẵn không giống dev board trống. Nó đã có firmware vendor, partition riêng, config sản xuất, có thể có calibration, key, certificate, model file, hoặc asset trong flash. Nếu bạn flash bừa, bạn có thể xóa thứ không tải lại được từ internet. Vì vậy bước học đầu tiên với DS-02 là quan sát và backup, không phải sửa.

Observe nghĩa là cắm serial, capture boot log, ghi reset reason, chip revision, flash mode/size, PSRAM, firmware string nếu có. Backup nghĩa là đọc full flash từ offset 0 tới hết dung lượng thật và lưu file rõ ràng. Identify nghĩa là suy ra partition, peripheral, pin map từ log, PCB, I2C scan, code tham khảo, không đoán một nguồn duy nhất là đúng.

Khi đọc xiaozhi-esp32, đừng đọc từ file ngẫu nhiên. Đọc theo runtime path: entry/state machine trước, board config sau, audio pipeline sau nữa, protocol/network, cuối cùng display/UI. Mục tiêu là trả lời “thiết bị này boot xong tạo task nào, task nào nói chuyện với peripheral nào, data audio đi qua những buffer nào”.

Điều kiện được flash lab nhỏ lên DS-02 phải rất rõ: đã có full backup, đã biết flash size thật, đã giữ được boot log gốc, biết vùng nào lab sẽ ghi đè, và chấp nhận cách restore. Nếu một điều kiện chưa đạt, lab nên chạy trên dev board hoặc dừng lại ở observe-only.

DS-02 nên được học như một hệ thống thật, không phải như board ESP32 trống. Bạn không biết vendor đã lưu gì trong NVS, filesystem hoặc partition riêng. Bạn cũng chưa chắc pinout public khớp đúng revision trên tay. Vì vậy method đúng là tạo hồ sơ thiết bị trước: ảnh board nếu có, serial log, flash id, backup hash, partition nghi ngờ, peripheral nghi ngờ, và danh sách vùng không được ghi.

Khi đọc code tham khảo như xiaozhi-esp32, câu hỏi không phải “file nào thú vị nhất?”. Câu hỏi là runtime: boot xong state machine nào chạy, task nào được tạo, board config nào chọn pin, audio đi qua buffer/codec nào, network protocol đổi state UI ra sao. Lesson này dạy cách đi từ quan sát thiết bị thật sang đọc code có trật tự.

Checklist trước khi flash bất cứ thứ gì

Path đọc code Xiaozhi sau khi có nền

  1. Tìm entry point và app state machine.
  2. Tìm board config: pin LCD, touch, codec, I2S, ADC, button.
  3. Tìm audio pipeline: mic capture, encode, network, decode, speaker.
  4. Tìm protocol: WebSocket/MQTT, activation, session state.
  5. Tìm display layer: emoji/state/UI update.
  6. Chỉ sau đó mới nghĩ tới custom board config cho DS-02.
Kết quả mong muốn không phải “có firmware mới ngay”, mà là bạn đọc được boot log, hiểu map file, biết vì sao có nhiều partition, biết task nào làm gì, và có thể tự thiết kế lab kiểm chứng từng phần cứng.
